Wazuh on Proxmox Installation Guide

Step-by-step instructions to install and configure Wazuh as a virtual machine on Proxmox VE

  1. Open any browser and search for WAZUH installation

    Navigate to the official Wazuh website to begin the installation process.

    Note: Ensure you have a stable internet connection for downloading required files.

  2. Click Installation Alternatives > click virtual machine (OVA)

    Select the OVA option for virtual machine deployment.

  3. Copy the OVA file link

    On the package - Right click on wazuh-4.13.0.ova and select "Copy link address".

  4. Access Proxmox server shell

    Open your PROXMOX server > select the node > select shell.

  5. Create a folder for the OVA file

    Navigate to /var/lib/vz and create a folder for the .ova file container:

    cd /var/lib/vz mkdir wazuh cd wazuh

  6. Download the Wazuh OVA package

    Run the wget command to download the OVA file:

    wget https://packages.wazuh.com/4.x/vm/wazuh-4.13.0.ova

    Wait until the package is fully downloaded.

  7. Extract the OVA file

    Extract the OVA file contents using tar command:

    tar -xvf wazuh-4.13.0.ova

    This will extract 3 files with different extensions: .ovf, .vmdk, .mf

  8. Create a new VM in Proxmox

    On your PROXMOX Server, create a new virtual machine with ID 300.

    Note: You can name it "wazuh-server" for easy identification.

  9. Import the disk to Proxmox

    Run the qm importdisk command to import the VMDK file:

    qm importdisk 300 /var/lib/vz/wazuh/wazuh-4.13.0-disk-1.vmdk local-lvm

  10. Attach the disk and configure boot order

    Once the import is complete:

    • Select the VM 300 > Hardware > Attach the disk
    • Options > Boot order > Edit > Select the disk

  11. Start the Wazuh server

    Start the virtual machine from the Proxmox interface.

  12. Access the Wazuh web interface

    Open your browser and navigate to the Wazuh server IP address.

  13. Login to Wazuh

    Use the default credentials to log in:

    Username: admin Password: admin
    Security Note: Change the default password after first login.

  14. Deploy a new agent

    On the dashboard, click "Deploy new agent" to set up monitoring for a client machine.

  15. Configure the agent

    Follow the steps in the agent deployment wizard, providing all required information.

  16. Copy the installation command

    Copy the command generated from the Wazuh dashboard for agent installation.

  17. Install the agent on Windows client

    Open PowerShell or Command Prompt with admin rights, then paste and run the command:

    Invoke-WebRequest -Uri https://packages.wazuh.com/4.x/windows/wazuh-agent-4.13.0-1.msi -OutFile $env:tmp\wazuh-agent; msiexec.exe /i $env:tmp\wazuh-agent /q WAZUH_MANAGER='192.168.0.50' WAZUH_AGENT_GROUP='default' WAZUH_AGENT_NAME='Windows11_Client'
    Note: Replace the IP address and agent name with your specific values.

  18. Start the Wazuh agent service

    Run the following command to start the agent:

    NET START Wazuh

  19. Verify agent connection

    Wait until the client appears on the Wazuh dashboard, indicating successful connection.

Prerequisites

  • Proxmox VE installed and running
  • Sufficient storage space (at least 20GB recommended)
  • Adequate RAM (8GB minimum, 16GB recommended)
  • Network connectivity between Proxmox and client machines
  • Root or sudo access to Proxmox server

Troubleshooting Tips

  • Check firewall settings if agents can't connect
  • Verify VM resources if Wazuh runs slowly
  • Check logs in /var/ossec/logs for errors
  • Ensure time synchronization between server and agents
